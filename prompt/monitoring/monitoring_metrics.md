## 一、文档说明

### 1.1 填写说明

| 字段 | 说明 |
|------|------|
| **采集粒度** | 该指标在哪个层级采集（Node / Pod / Container / Process / GPU / NIC / Switch） |
| **采集频率** | 实时监控间隔（如 1s / 5s / 15s / 60s / 事件触发） |
| **聚合方式** | 是否需要跨节点聚合、是否需要时间窗口聚合 |
| **告警策略** | 建议的阈值类型（固定阈值 / 同比 / 环比 / 基线偏离 / 事件触发） |
| **适用场景** | 预训练 / 微调 / RLHF / 推理 / 全场景 |

### 1.2 五层架构概览

```
┌─────────────────────────────────────────┐
│  五、业务与成本层（Business & Cost）      │  ← 成本核算、碳排放、SLA、队列效率
├─────────────────────────────────────────┤
│  四、任务/模型层（Job & Model）           │  ← 任务生命周期、收敛性、资源画像
├─────────────────────────────────────────┤
│  三、训练框架层（Framework）               │  ← 分布式通信、并行策略、检查点、MFU
├─────────────────────────────────────────┤
│  二、系统运行时层（OS & Runtime）         │  ← K8s 容器、进程状态、驱动、电源
├─────────────────────────────────────────┤
│  一、基础设施硬件层（Hardware）             │  ← GPU/CPU/网络/存储/温度/功耗
└─────────────────────────────────────────┘
```

## 九、指标优先级与采集方式建议

### 9.1 优先级定义

| 优先级 | 定义 | 响应时效 |
|:---|:---|:---|
| **P0（核心）** | 故障定位、任务存活、基础利用率 | 实时（< 1s） |
| **P1（效率）** | 性能瓶颈分析、成本核算 | 近实时（< 15s） |
| **P2（优化）** | 精细化调优、容量规划 | 分钟级（< 5min） |
| **P3（合规）** | 审计、安全、ESG | 小时级（< 1h） |

### 9.2 采集方式备注

建议工程师在补充指标时注明以下信息：

| 字段 | 说明 |
|:---|:---|
| **指标来源** | DCGM / NVML / PyTorch Profiler / NCCL Test / eBPF / cgroup / Prometheus Exporter / 自定义 Hook |
| **性能开销** | 是否会对训练性能产生副作用（如 PyTorch Profiler 开销约 5-10%） |
| **处理方式** | 是否需要离线聚合还是实时流处理 |
| **依赖版本** | 特定驱动版本、框架版本或内核版本要求 |

### 9.3 技术栈适配建议

根据实际技术栈裁剪或增强以下类别：

| 技术栈 | 建议关注大类 |
|:---|:---|
| Kubernetes 集群 | 3.1 容器/虚拟化、6.2 队列/调度 |
| Megatron-LM / DeepSpeed / FSDP | 4.1 并行策略、4.2 计算效率 |
| RoCE 网络 | 2.2.2 RoCE/IB、4.1.1 NCCL |
| InfiniBand 网络 | 2.2.2 RoCE/IB、4.1.1 NCCL |
| 液冷数据中心 | 2.1.1 GPU 温度/功耗、3.3.2 电源/散热 |
| 多租户平台 | 7.2 多租户/隔离、6.1 成本核算 |
